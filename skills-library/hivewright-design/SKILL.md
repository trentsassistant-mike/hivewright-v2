If creating visual artifacts (slides, mocks, throwaway prototypes, etc), copy assets out and create static HTML files for the user to view. If working on production code, you can copy assets and read the rules here to become an expert in designing with this brand.
If the user invokes this skill without any other guidance, ask them what they want to build or design, ask some questions, and act as an expert designer who outputs HTML artifacts _or_ production code, depending on the need.

## Quick reference
- **Tokens:** `colors_and_type.css` — drop into any HTML, all CSS vars + `.hw-*` classes work immediately.
- **Brand:** `assets/brand/` — the hex H mark (`hivewright_mark.svg`), mono variant, lockup, hex grid background, mood board.
- **Icons:** `assets/icons/icons.svg` — fine-line construction set, `<use href="...#name"/>`.
- **UI Kit:** `ui_kits/app/` — full PWA dashboard recreation (Sidebar, TopBar, KpiCard, OperationsMap, AgentList, RunsTable, Suggestions, WorkflowCanvas, NewAutomationModal). Open `ui_kits/app/index.html` to see it run.
- **Voice:** operational, calm, no hype, no emoji. See README CONTENT FUNDAMENTALS for examples.
- **Don't:** cartoon bees, honey-jar/farm vibes, purple AI gradients, robot imagery, pure white text, multi-element honey glows.
- **Do:** graphite surfaces · molten honey accents · honeycomb structural geometry · sage/ember signal accents · warm ivory text.
